## Руководство по эксплуатации API сервиса для социальной сети

Сервис создан на языке программирования python и использует базу данных Postgres для хранения данных пользователей, сообщений, лайков и подписок

### Описание работы

В данном сервисе используется фреймворк FastApi, осуществляющий асинхронное взаимодействие с бд Postgres. Веб-сервер реализован на nginx.
Все сервисы запускаются в составном докер-контейнере.

#### 1. Запуск сервиса

В консоли из корня проекта вводится команда, после запуска которой сервис, а так же и веб-приложение становятся доступны через порт 80:
```
docker-compose up -d
```

#### 2. Эндпоинты
1. **_/api/tweets_** method GET - Получить ленту пользователя. 
2. **_/api/users/me_** method GET  - Получить данные своего профиля.
3. **_/api/users/{id}_** method GET - Получить данные о произвольном профиле с указанным id
5. **_/api/users/{id}/follow_** method POST - Подписка на пользователя с указанным id
6. **_/api/users/{id}/follow_** method DELETE - Отмена подписки на пользователя с указанным id
7. **_/api/tweets/{id}/likes_** method POST - Установка/снятие лайка твиту с указанным id
8. **_/api/make_user_** method POST - Создание пользователя
9. **_/api/delete_user/{id}_** - method DELETE - Удаление пользователя
10. **_/api/medias_** method POST - Отправка файла.
11. **_/api/tweets_** method POST - Отправка твита.
12. **_/api/tweets/{id}_** method DELETE - Удаление твита с указанным id


#### 4. Настройка.
Данные аккаунта для базы данных (login, password), токен для эндпоинтов (token) передаются приложению через файл переменных окружения ./main/.env
В корне проекта так же расположен файл ./.env, в котором задаётся аккаунт для входа в базу данных и её название (db_login, db_password, db_name)


#### 3. Документация
Подробное описание эндпоинтов (swagger) с возможностью их тестирования доступна через порт 8000 по адресу:
```
 /docs
 ```
